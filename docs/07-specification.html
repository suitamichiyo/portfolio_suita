1 システム概要

システムの目的 必須
このシステムが解決する課題と提供する価値

inimuの運営者・スタッフが利用し、予約・顧客・問い合わせ情報を一元管理できる管理システムである。店舗運営の効率化と対応品質の安定化を図り、来店体験の向上と継続的な集客につなげる価値を提供する。

4 機能詳細仕様

F-001 ログイン認証

画面レイアウト

中央にロゴ、タイトル、メールアドレス入力、パスワード入力、エラーメッセージ、ログインボタンを配置する。

入力項目一覧
項目名データ型必須バリデーション備考
メールアドレスString○必須、メール形式ログインID
パスワードString○必須、8文字以上マスク表示
処理フロー / 動作 / 遷移

ユーザーが「ログインする」を押下すると、Spring Security が認証を行う。正常時はダッシュボードへ遷移する。入力不備時はインラインエラー、認証失敗時は「メールアドレスまたはパスワードが正しくありません」を表示する。システムエラー時は再試行メッセージを表示する。遷移先：ダッシュボード / 同画面。

F-002 予約一覧表示

画面レイアウト

パンくず、ページタイトル、新規登録ボタン、検索条件、件数表示、CSVボタン、一覧テーブル、ページネーションを配置する。

入力項目一覧
項目名データ型必須バリデーション備考
日付FromDate—日付形式検索条件
日付ToDate—日付形式、From以降検索条件
ステータスEnum—選択値チェック検索条件
氏名String—部分一致、100文字以内検索条件
処理フロー / 動作 / 遷移

一覧画面表示時に予約データを取得して表示する。正常時はテーブルと件数を表示する。検索条件不正時は条件欄にエラーを表示する。システムエラー時は一覧取得失敗メッセージを表示する。遷移先：「詳細」で予約詳細、「+新規登録」で予約新規登録。

F-003 予約新規登録

画面レイアウト

予約情報ブロック、お客様情報ブロック、確認メール送信選択、キャンセルボタン、登録ボタンを配置する。

入力項目一覧
項目名データ型必須バリデーション備考
予約日Date○必須、営業日チェック土日祝のみ
時間帯Enum○必須、空き枠チェック11/13/15時帯
参加人数Integer○必須、1〜上限定員料金自動計算
氏名String○必須、100文字以内—
ふりがなString○必須、ひらがな—
メールString○必須、メール形式—
電話番号String○必須、電話番号形式—
備考String—500文字以内—
処理フロー / 動作 / 遷移

入力後に「予約を登録する」を押下すると、空き枠確認、料金計算、予約登録、必要に応じて確認メール送信を行う。正常時は予約詳細へ遷移する。入力エラー時は各項目下に表示し、満席時は登録不可メッセージを出す。システムエラー時は登録失敗を通知する。遷移先：予約詳細 / 予約一覧。

画面レイアウト

予約番号、編集ボタン、一覧に戻るボタン、ステータスバッジ、予約情報、お客様情報、スタッフメモ、ステータス変更ボタンを配置する。

入力項目一覧
項目名データ型必須バリデーション備考
スタッフメモString—1000文字以内保存可能
ステータス変更Enum—状態遷移チェック入場済み / キャンセル
処理フロー / 動作 / 遷移

詳細表示時に予約情報と関連顧客情報を取得する。正常時は詳細情報を表示し、メモ保存やステータス変更を反映する。対象データ不存在時は一覧へ戻すかエラー表示する。システムエラー時は取得失敗を表示する。遷移先：予約編集 / 予約一覧 / 顧客詳細。

5 非機能要件

セキュリティ要件 必須
認証・認可・データ保護に関する要件
項目要件
認証方式Spring Security を用いたフォームログイン認証を採用し、未認証ユーザーは `/admin/login` へリダイレクトする。
パスワードの暗号化方式パスワードは平文保存せず、`BCryptPasswordEncoder` によりハッシュ化して保存する。
CSRF対策Spring Security の CSRF 保護を有効化し、状態変更を伴う POST/更新系リクエストに CSRF トークンを必須とする。
XSS対策Thymeleaf の自動エスケープを基本とし、入力値表示時は HTML エスケープを行う。危険なスクリプト文字列はそのまま出力しない。
SQLインジェクション対策Repository / ORM / パラメータバインドを用いて SQL を組み立て、文字列連結による生SQL埋め込みを避ける。
セッション管理ログイン後はサーバー側セッションで認証状態を管理し、ログアウト時にセッションを破棄する。セッションタイムアウトも設定する。
対応環境 必須
対応ブラウザ・デバイス

・対応ブラウザ：Chrome（最新版）/ Firefox（最新版）/ Edge（最新版）
・対応デバイス：PC / タブレット / スマートフォン
・推奨画面幅：PC 1440px基準 / Tablet 768〜1139px / Mobile 〜767px
