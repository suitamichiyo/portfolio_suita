1 パッケージ構成

ディレクトリ構造 必須
src/main/java以下のパッケージ構造
src/main/java/
└── com.example.taskapp
    ├── TaskAppApplication.java        # Spring Boot起動クラス
    ├── config/
    │   ├── SecurityConfig.java        # Spring Security設定
    │   └── WebConfig.java             # MVC共通設定
    ├── controller/
    │   ├── LoginController.java       # ログイン画面/認証関連
    │   ├── TaskController.java        # タスク一覧・詳細・登録・更新・削除
    │   └── UserController.java        # ユーザー管理
    ├── service/
    │   ├── TaskService.java           # タスク業務ロジック
    │   └── UserService.java           # ユーザー業務ロジック
    ├── repository/
    │   ├── TaskMapper.java            # MyBatisによるtasksテーブルアクセス
    │   └── UserMapper.java            # MyBatisによるusersテーブルアクセス
    ├── entity/
    │   ├── Category.java              # カテゴリ情報
    │   ├── Task.java                  # タスク情報
    │   └── User.java                  # ユーザー情報
    └── form/
        ├── LoginForm.java             # ログイン入力フォーム
        └── TaskForm.java              # タスク登録・編集フォーム
パッケージの役割 必須
各パッケージの責務を説明
パッケージ 役割 使用するアノテーション
config セキュリティ設定やMVC共通設定など、アプリケーション全体の設定を管理する。 @Configuration, @EnableWebSecurity
controller HTTPリクエストを受け取り、Serviceを呼び出して画面名やリダイレクト先を返却する。 @Controller, @GetMapping, @PostMapping
service 入力値の業務チェック、エンティティ変換、登録・更新・検索などの業務ロジックを実装する。 @Service, @Transactional
repository MyBatis MapperとしてSQLを実行し、MySQLとのデータ入出力を担当する。 @Mapper
entity データベーステーブルに対応する業務データのモデルを定義する。 @Data, @NoArgsConstructor
form 画面入力値を受け取り、Bean Validationで入力チェックを行う。 @Data, @NotBlank, @Size

3 Repository（リポジトリ）

TaskMapper.java 必須
タスクリポジトリのインターフェース定義
┌────────────────────────────────────────────────┐
│              《Interface》                      │
│               TaskMapper                        │
│              @Mapper                            │
├────────────────────────────────────────────────┤
│ + selectAll(): List<Task>                     │
│ + selectById(id: Long): Task                   │
│ + selectByUserId(userId: Long): List<Task>    │
│ + insert(task: Task): void                     │
│ + update(task: Task): void                     │
│ + delete(id: Long): void                       │
│ + search(keyword: String): List<Task>         │
└────────────────────────────────────────────────┘
メソッド 引数 戻り値 SQL種別 説明
selectAll なし List<Task> SELECT tasksテーブルからタスク情報を全件取得する。
selectById id: Long Task SELECT 主キーIDを条件に1件のタスク情報を取得する。
selectByUserId userId: Long List<Task> SELECT 指定ユーザーに紐づくタスク一覧を取得する。
insert task: Task void INSERT Taskエンティティの内容をtasksテーブルへ新規登録する。
update task: Task void UPDATE 主キーIDをもとに既存タスク情報を更新する。
delete id: Long void DELETE 主キーIDを指定してタスク情報を削除する。
search keyword: String List<Task> SELECT タイトルや内容を対象にキーワード検索した結果を取得する。

4 Service（サービス）

TaskService.java 必須
タスクサービスのクラス定義
┌────────────────────────────────────────────────────┐
│                   《Service》                       │
│                   TaskService                       │
│             @Service / @Transactional               │
├────────────────────────────────────────────────────┤
│ - taskMapper: TaskMapper                           │
├────────────────────────────────────────────────────┤
│ + findAll(): List<Task>                           │
│ + findById(id: Long): Task                         │
│ + findByUserId(userId: Long): List<Task>          │
│ + create(form: TaskForm, userId: Long): void       │
│ + update(id: Long, form: TaskForm): void           │
│ + delete(id: Long): void                           │
│ + search(keyword: String): List<Task>             │
│ + changeStatus(id: Long, status: String): void     │
└────────────────────────────────────────────────────┘

依存関係：
  TaskService ────▶ TaskMapper （@Autowired）

FormからEntityへの変換処理：
  create / update では TaskForm から入力値を受け取り、
  Task エンティティへ値を詰め替えて Mapper に渡す。
  変換方法は BeanUtils.copyProperties または setter による手動マッピングを使用する。
  例：
    - form.title      -> task.title
    - form.content    -> task.content
    - form.dueDate    -> task.dueDate
    - form.priority   -> task.priority
    - form.status     -> task.status
    - userId          -> task.userId

@Transactional の使用箇所：
  - create：登録処理を1トランザクションで実行
  - update：更新処理を1トランザクションで実行
  - delete：削除処理を1トランザクションで実行
  - changeStatus：状態変更を1トランザクションで実行
項目 内容
依存するMapper TaskMapper をDIし、タスク情報の取得・登録・更新・削除をRepository層へ委譲する。
CRUDメソッド findAll, findById, create, update, delete により、タスクの参照・登録・更新・削除を行う。
検索メソッド search(keyword) でタイトルや内容を対象としたキーワード検索を行う。
ステータス変更メソッド changeStatus(id, status) で対象タスクの状態のみを変更する。
Form→Entity変換 TaskForm の各入力項目を Task へ変換して保存する。必要に応じて BeanUtils.copyProperties や setter による手動マッピングを利用する。
@Transactional create, update, delete, changeStatus など、データ更新を伴うメソッドに適用する。

5 Controller（コントローラー）

TaskController.java 必須
タスクコントローラーのクラス定義
┌─────────────────────────────────────────────────────────┐
│                    《Controller》                        │
│                    TaskController                        │
│                    @Controller                           │
│                    @RequestMapping("/tasks")             │
├─────────────────────────────────────────────────────────┤
│ - taskService: TaskService                              │
├─────────────────────────────────────────────────────────┤
│ + list(model: Model): String                            │
│ + show(id: Long, model: Model): String                  │
│ + createForm(model: Model): String                      │
│ + create(form: TaskForm, result: BindingResult): String │
│ + editForm(id: Long, model: Model): String              │
│ + update(id: Long, form: TaskForm, result: BindingResult): String │
│ + delete(id: Long): String                              │
└─────────────────────────────────────────────────────────┘
メソッド HTTP URL 説明
list GET /tasks タスク一覧画面を表示する。
show GET /tasks/{id} 指定IDのタスク詳細画面を表示する。
createForm GET /tasks/new タスク新規作成フォームを表示する。
create POST /tasks フォーム入力値を受け取り、新規タスクを登録する。
editForm GET /tasks/{id}/edit 指定IDのタスク編集フォームを表示する。
update POST /tasks/{id} 指定IDのタスクを更新する。
delete POST /tasks/{id}/delete 指定IDのタスクを削除する。
