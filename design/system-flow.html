1 システム全体構成

システム構成図 必須
Spring Bootアプリケーションのレイヤー構成
+-------------------------+      HTTP Request       +-------------------------------------------+
| クライアント            | ----------------------> | Spring Boot Application                   |
| ブラウザ                |                         |                                           |
+-------------------------+                         |  +---------------+                        |
          ^                                        |  | Controller    |                        |
          |                                        |  +-------+-------+                        |
          | HTMLレスポンス                         |          |                                |
          |                                        |          v                                |
          |                                        |  +---------------+                        |
          |                                        |  | Service       |                        |
          |                                        |  +-------+-------+                        |
          |                                        |          |                                |
          |                                        |          v                                |

          |                                        |  +-------+-------+                        |

                 HTML描画結果                         | View         |                        |
                                                       +-------+-------+                        |
                                                               ^                                |
                                                               | Modelデータ                    |
                                                               +--------------------------------+
レイヤー構成の説明 必須
各レイヤーの役割を表形式で整理する
レイヤー 役割 データの流れ
クライアント（ブラウザ） ユーザー操作、画面表示、フォーム送信を行う HTTPリクエスト送信、HTMLレスポンス受信
Controller層 リクエスト受付、入力値受取、Service呼び出し、ビュー返却を行う ブラウザ → Controller → Service
Service層 業務ロジック、入力値整形、処理制御を行う Controller → Service → Repository
Repository層（MyBatis Mapper） SQL実行とデータベースアクセスを担当する Service → Repository → MySQL
データベース（MySQL） 予約、顧客、問い合わせ、ユーザーなどのデータを永続化する Repository に検索結果や更新結果を返す
Thymeleafビュー Controller から受け取った Model データを HTML に反映して画面を生成する Controller → Thymeleaf → ブラウザ

5 認証・認可フロー

認証フロー 必須
Spring Security による基本的な認証の流れを示す
ブラウザ
  |
  | POST /login
  v
Spring Security Filter
  |
  v
Authentication Manager
  |
  v
UserDetailsService
  |
  v
UserMapper
  |
  v
MySQL
  |
  +--> 認証成功: /tasks へリダイレクト
  +--> 認証失敗: /login?error を表示
アクセス制御一覧 任意
URLごとのアクセス可否の例
機能 URL ADMIN USER 未認証
ログイン /login 可 可 可
タスク一覧 /tasks 可 可 不可
タスク新規登録 /tasks/new 可 可 不可
顧客管理 /customers 可 可 不可
管理者設定 /admin/settings 可 不可 不可
