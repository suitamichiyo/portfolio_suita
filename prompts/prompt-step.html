Entity作成
DB設計書を元にEntityクラスを作成する

前提条件

  • DB設計書が完成していること
  • MySQLにデータベースとテーブルが作成済みであること

入力プロンプト例

以下のDB設計書を元に、Entityクラスを作成してください。

【テーブル定義】
■ tasks テーブル
| カラム名     | 型           | NULL | 説明                 |
|-------------|--------------|------|----------------------|
| id          | BIGINT       | NO   | 主キー（自動採番）    |
| title       | VARCHAR(100) | NO   | タイトル              |
| content     | TEXT         | YES  | 内容                 |
| due_date    | DATE         | YES  | 期限                 |
| priority    | INT          | NO   | 優先度（1-3）         |
| status      | VARCHAR(20)  | NO   | ステータス            |
| user_id     | BIGINT       | NO   | ユーザーID（外部キー） |
| created_at  | DATETIME     | NO   | 作成日時              |
| updated_at  | DATETIME     | NO   | 更新日時              |

【要件】
・パッケージ：com.example.taskapp.entity
・Lombokアノテーション使用（@Data, @NoArgsConstructor, @AllArgsConstructor）
・日付型はLocalDate、日時型はLocalDateTimeを使用
・フィールド名はキャメルケース、カラム名はスネークケースで対応

Task.java 出力例

package com.example.taskapp.entity;

import java.time.LocalDate;
import java.time.LocalDateTime;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@Data
@NoArgsConstructor
@AllArgsConstructor
public class Task {

    private Long id;
    private String title;
    private String content;
    private LocalDate dueDate;
    private Integer priority;
    private String status;
    private Long userId;
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
}

補足

  • `due_date` は `dueDate`、`user_id` は `userId`、`created_at` は `createdAt` のようにキャメルケースへ変換する
  • `DATE` は `LocalDate`、`DATETIME` は `LocalDateTime` に対応させる
  • MyBatisの `map-underscore-to-camel-case=true` を使うとスネークケースとキャメルケースを自動対応しやすい

期待される出力

  • `src/main/java/com/example/taskapp/entity/Task.java`
  • Lombokアノテーション付きのEntityクラス
  • DBカラムとJavaフィールド対応が分かるコード

確認ポイント

  • フィールドの型が適切か（Long, String, LocalDate等）
  • Lombokアノテーションが正しく設定されているか
  • スネークケースのカラム名とキャメルケースのフィールド名が対応しているか
03
Repository作成
MyBatis Mapperインターフェースを作成する

入力プロンプト例

TaskエンティティのMyBatis Mapperを作成してください。

【要件】
・パッケージ：com.example.taskapp.repository
・インターフェース名：TaskMapper
・@Mapperアノテーションを使用

【実装するメソッド】
1. selectAll() - 全件取得（作成日時降順）
2. selectById(Long id) - ID指定取得
3. selectByUserId(Long userId) - ユーザー別取得
4. insert(Task task) - 新規登録（自動採番のIDを取得）
5. update(Task task) - 更新
6. delete(Long id) - 削除

【SQL実装方式】
・アノテーション方式（@Select, @Insert, @Update, @Delete）を使用
・カラム名とフィールド名のマッピングは@Resultsを使用

【注意点】
・insertではuseGeneratedKeys=trueを指定してIDを取得
・キャメルケース⇔スネークケースの変換に対応

TaskMapper.java 出力例

package com.example.taskapp.repository;

import java.util.List;

import org.apache.ibatis.annotations.Delete;
import org.apache.ibatis.annotations.Insert;
import org.apache.ibatis.annotations.Mapper;
import org.apache.ibatis.annotations.Options;
import org.apache.ibatis.annotations.Result;
import org.apache.ibatis.annotations.Results;
import org.apache.ibatis.annotations.Select;
import org.apache.ibatis.annotations.Update;

import com.example.taskapp.entity.Task;

@Mapper
public interface TaskMapper {

    @Results(id = "taskResultMap", value = {
        @Result(property = "id", column = "id"),
        @Result(property = "title", column = "title"),
        @Result(property = "content", column = "content"),
        @Result(property = "dueDate", column = "due_date"),
        @Result(property = "priority", column = "priority"),
        @Result(property = "status", column = "status"),
        @Result(property = "userId", column = "user_id"),
        @Result(property = "createdAt", column = "created_at"),
        @Result(property = "updatedAt", column = "updated_at")
    })
    @Select("""
        SELECT id, title, content, due_date, priority, status, user_id, created_at, updated_at
        FROM tasks
        ORDER BY created_at DESC
        """)
    List<Task> selectAll();

    @Select("""
        SELECT id, title, content, due_date, priority, status, user_id, created_at, updated_at
        FROM tasks
        WHERE id = #{id}
        """)
    @org.apache.ibatis.annotations.ResultMap("taskResultMap")
    Task selectById(Long id);

    @Select("""
        SELECT id, title, content, due_date, priority, status, user_id, created_at, updated_at
        FROM tasks
        WHERE user_id = #{userId}
        ORDER BY created_at DESC
        """)
    @org.apache.ibatis.annotations.ResultMap("taskResultMap")
    List<Task> selectByUserId(Long userId);

    @Insert("""
        INSERT INTO tasks (title, content, due_date, priority, status, user_id, created_at, updated_at)
        VALUES (#{title}, #{content}, #{dueDate}, #{priority}, #{status}, #{userId}, #{createdAt}, #{updatedAt})
        """)
    @Options(useGeneratedKeys = true, keyProperty = "id")
    int insert(Task task);

    @Update("""
        UPDATE tasks
        SET title = #{title},
            content = #{content},
            due_date = #{dueDate},
            priority = #{priority},
            status = #{status},
            user_id = #{userId},
            updated_at = #{updatedAt}
        WHERE id = #{id}
        """)
    int update(Task task);

    @Delete("""
        DELETE FROM tasks
        WHERE id = #{id}
        """)
    int delete(Long id);
}

補足

  • `due_date` → `dueDate`、`user_id` → `userId`、`created_at` → `createdAt` を `@Results` で対応させる
  • `insert` は `@Options(useGeneratedKeys = true, keyProperty = "id")` で自動採番IDを取得する
  • 戻り値を `int` にすると、更新件数や削除件数も確認しやすい

期待される出力

  • `src/main/java/com/example/taskapp/repository/TaskMapper.java`
  • `@Mapper` 付きのMyBatisインターフェース
  • CRUDメソッドと `@Results` によるマッピング定義

確認ポイント

  • @Mapperアノテーションが付与されているか
  • SQLの構文が正しいか
  • カラム名とフィールド名のマッピングが正しいか
  • `insert` で採番されたIDが `task.id` に格納される設定になっているか
04
Service作成
ビジネスロジック層のServiceクラスを作成する

入力プロンプト例

TaskServiceクラスを作成してください。

【要件】
・パッケージ：com.example.taskapp.service
・クラス名：TaskService
・@Serviceアノテーションを使用
・TaskMapperをコンストラクタインジェクション

【実装するメソッド】
1. findAll() - 全タスク取得
2. findById(Long id) - ID指定取得
3. findByUserId(Long userId) - ユーザー別取得
4. create(TaskForm form, Long userId) - 新規作成
   - FormからEntityへの変換処理
   - 作成日時・更新日時の設定
5. update(Long id, TaskForm form) - 更新
   - 既存データの取得と更新
   - 更新日時の設定
6. delete(Long id) - 削除

【トランザクション】
・create, update, deleteメソッドには@Transactionalを付与

【Form → Entity 変換】
・TaskForm（画面入力）からTask（エンティティ）への変換ロジックを含める

TaskService.java 出力例

package com.example.taskapp.service;

import java.time.LocalDateTime;
import java.util.List;

import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import com.example.taskapp.entity.Task;
import com.example.taskapp.form.TaskForm;
import com.example.taskapp.repository.TaskMapper;

@Service
public class TaskService {

    private final TaskMapper taskMapper;

    public TaskService(TaskMapper taskMapper) {
        this.taskMapper = taskMapper;
    }

    public List<Task> findAll() {
        return taskMapper.selectAll();
    }

    public Task findById(Long id) {
        return taskMapper.selectById(id);
    }

    public List<Task> findByUserId(Long userId) {
        return taskMapper.selectByUserId(userId);
    }

    @Transactional
    public void create(TaskForm form, Long userId) {
        Task task = new Task();
        LocalDateTime now = LocalDateTime.now();

        task.setTitle(form.getTitle());
        task.setContent(form.getContent());
        task.setDueDate(form.getDueDate());
        task.setPriority(form.getPriority());
        task.setStatus(form.getStatus());
        task.setUserId(userId);
        task.setCreatedAt(now);
        task.setUpdatedAt(now);

        taskMapper.insert(task);
    }

    @Transactional
    public void update(Long id, TaskForm form) {
        Task task = taskMapper.selectById(id);
        if (task == null) {
            throw new IllegalArgumentException("指定されたタスクが存在しません");
        }

        task.setTitle(form.getTitle());
        task.setContent(form.getContent());
        task.setDueDate(form.getDueDate());
        task.setPriority(form.getPriority());
        task.setStatus(form.getStatus());
        task.setUpdatedAt(LocalDateTime.now());

        taskMapper.update(task);
    }

    @Transactional
    public void delete(Long id) {
        taskMapper.delete(id);
    }
}

補足

  • 読み取り系の `findAll` `findById` `findByUserId` は通常 `@Transactional` 不要
  • `create` では `createdAt` と `updatedAt` を同じ現在時刻で設定する
  • `update` では既存データを取得してから変更対象フィールドのみ上書きする
  • `TaskForm → Task` 変換は setter による手動マッピングで表現している

期待される出力

  • `src/main/java/com/example/taskapp/service/TaskService.java`
  • `@Service` 付きのサービスクラス
  • CRUDメソッドと `TaskForm → Task` 変換ロジック

確認ポイント

  • `TaskMapper` がコンストラクタインジェクションされているか
  • `create` `update` `delete` に `@Transactional` が付与されているか
  • `TaskForm` の入力値が `Task` に正しく詰め替えられているか
  • `createdAt` `updatedAt` が適切に設定されているか
